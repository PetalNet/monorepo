Layer 1: Sensor Gate Research-Backed

The accelerometer is 60x cheaper than GPS. Use it as a gatekeeper.

→
GPS hardware stays OFF by default. The accelerometer (5-10mW) watches for motion. Only when it detects significant movement does GPS (200-300mW) power on. People are stationary 80%+ of the day.
→
Android: TYPE_SIGNIFICANT_MOTION sensor — hardware-accelerated, fires once on movement, costs zero battery. Same mechanism Google Maps uses.
→
iOS: CMMotionActivityManager for activity classification at near-zero cost.
→
Hysteresis: 10s sustained movement before waking GPS. 30s no movement before killing GPS. Prevents thrashing.
Why: A GPS fix costs ~300mJ. An accelerometer read costs ~5mJ. By gating GPS, we eliminate 80%+ of wasted fixes. Life360 polls GPS on a fixed timer regardless.

Layer 2: Auto Hidden Zones NEW

Learn where users spend time. When they're there, GPS is unnecessary.

1
Detect frequent locations. 3+ visits of 30+ min at same spot (100m) = learned zone. Typical: home, work, school, gym.
2
Inside a zone: GPS off. Report zone center. WiFi/cell fingerprint confirms presence (100x less than GPS).
3
Exit detection: WiFi BSSID changes + accelerometer = you left. GPS wakes immediately. OS geofence registered for reliability.
4
Privacy: Zones stored only on-device. Never uploaded. Never shared. Not in MLS payload. Pure battery optimization.
5
Refinement: Radius shrinks with confidence. 30 days unvisited = pruned. User can view/delete in settings.
Impact: Average person spends 80% of their day in 2-3 locations. With auto-zones, GPS is off 80% of the day. Single biggest battery win.

Layer 4: Demand-Driven Updates

Don't poll constantly. Only send fresh data when someone wants it.

→
Nudge: Tap someone's marker → their device wakes GPS → one fix → sent back. <5s. Cost: one fix (~300mJ = nothing).
→
Adaptive rate: Sustained viewing: nudge 15s if moving, 45s if recent, 2min if stale.
→
Nobody watching + still = absolute zero. This is where we crush Life360.
→
App killed: FCM/UP wakes → one fix → relay → sleep. ~2s total wake.

Layer 5: Batched Network Relay Research-Backed

The radio costs 5-10x more than GPS. Batch fixes, send once.

→
Foreground: Relay on timer (3-5s). WS stays open — no radio wake cost.
→
Background: Accumulate fixes locally. Flush every 30s or on 5+ fixes. Single compressed burst.
→
WiFi opportunism: Flush immediately on WiFi. Less cost than cellular.
→
Compression: Delta-encode. Typical update <100 bytes. Daily total <500KB.
→
Server interpolation: Kalman filter between sparse bg fixes. Viewers see smooth movement from 10-15s data.

Auto Zone Detection

The phone learns your patterns. You never configure anything.

1
Dwell detection. Same location (100m) for 30+ min → recorded as dwell event. Local only.
2
Cluster → zone. 3+ dwells at same cluster → learned zone. Radius: 150m initial → 50m refined.
3
OS geofence. Top-5 zones get registered geofences. Survive app kill. Fire exit events reliably.
4
WiFi fingerprint. Same BSSIDs = still in zone. Cheap confirmation without GPS.
5
Pruning. 30 days unvisited → confidence drops → deleted. Stays current.
6
Privacy. Never leaves the device. Not in server DB, not in MLS payload, not in shared data.

Rendering Pipeline

1
GPS fix arrives → store as targetPosition. Marker doesn't jump.
2
Every 16ms (60fps) → lerp toward target at 30%/frame. Smooth glide.
3
Other people → same pipeline. WS pushes become targets. Even 15s bg fixes look smooth.
4
Trails → Catmull-Rom spline. Smooth curves from sparse data.
5
Camera → follows lerped pos via moveCamera. Only when following someone. Free pan otherwise.
6
Stale markers → >5min: dim 50%. >1hr: "last seen" label. Ghost: vanish.
